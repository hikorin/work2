# Mobile Access Setup 📱✨

## 概要
このスキルは、ローカルで開発している `kaeshi_app` にスマホやタブレットからアクセスできるようにするためのもんや。
Tailscale などの仮想ネットワーク経由での接続や、IPアドレスの自動検出、サーバーのホスト設定まで全部お任せやで！🔥

## ワークフロー (Workflow) 🛠️

モバイルからのアクセスをセットアップする時は、以下の手順で進めるんや：

1.  **接続確認 (Connection Check)** 🔍
    - `http://[IP]:5173` (フロントエンド) と `http://[IP]:8080` (バックエンド) に接続できるか確認する。
    - Tailscale環境を考慮し、まずは `100.98.193.61` などの優先IPをチェックするんやで！🚀

2.  **IPアドレスの特定 (IP Identification)** 🕵️‍♀️
    - スクリプトが `ipconfig` 実行結果から `100.98.193.61` などの優先IPが存在するかチェックし、見つからなければ `socket` モジュールでデフォルトIPを自動取得する。

3.  **設定ファイルの更新 (Update Config)** 📝
    - `kaeshi_app/frontend/src/components/*.tsx` などの API 接続先 URL を、特定した IP アドレス (ポート `8080`) に書き換える。
    - **注意点**: 以前の `8000` 番ポートはWindows環境でゴーストプロセス化（ゾンビ化）しやすいため、安全な `8080` 番を使うのが絶対の掟や！⚠️

4.  **サーバーの起動 (Start Servers)** 🚀
    - バックエンド (FastAPI): `0.0.0.0:8080` で起動する。
    - フロントエンド (Vite): `0.0.0.0:5173` で `--host` オプション付きで起動する。
    - **超重要**: Windowsプロンプトの文字コードエラー(cp932)による異常終了を防ぐため、実行時は必ず `$env:PYTHONIOENCODING="utf-8"` を頭につけてから `python setup_mobile.py` を走らせること！🔥

5.  **疎通確認 (Verification)** ✅
    - `netstat -an | findstr "8080 5173"` でポートがちゃんと `0.0.0.0` で開いてるか確認する。
    - スマホからアクセスして、バックエンドが繋がらへん場合は **CORS設定** を疑うべし！🕵️‍♀️✨

6.  **完了報告 (Output URLs)** 📢
    - ユーザーに最終的な URL をドヤ顔で教える！💖

## リソース (Resources) 📚

### scripts/
- `setup_mobile.py`: IP検出、URL置換、サーバー起動を自動化するスクリプト。

## トラブルシューティング 🛠️
- **フロントは見れるけどバックエンドがエラー**: `main.py` の `CORSMiddleware` にフロントエンドの IP:Port が許可されてるかチェック！
- **API が 404**: フロントエンドの `fetch` パスとバックエンドの `router` パス（`/api` の有無など）が一致してるか確認！
